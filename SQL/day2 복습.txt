 * 제약 조건 constraint - 테이블에 저장되는 데이터가 `요구사항에 대한 부적절한 값`을 갖지 않도록 규정.
 	ㄴ 테이블을 구성하는 컬럼에 설정합니다. 
 * 
 * - ★ 무결성 ( Integrity , 데이터 정확성/신뢰성/일관성) 유지. 
	컬럼에 부여하는 조건. ㄴ 관련 제약 조건 : UNIQUE , primary key(pk)기본키 ,  FOREIGN KEY (fk) 외래키
 * 
 * 1) NOT NULL - 반드시 값이 있어야한다.(필수입력)		NULL 허용안함. 

 * 2) UNIQUE (유일한) - 특정 컬럼이 같은 값을 가질수 없다.	- NULL 값은 가능함. 

	----> 오라클은 검색 성능을 위해서 index 오브젝트를 만든다. 검색 Index 를 만든다.

 * 3) CHECK - 예시로 입력값이 1~4 범위로만 한다. 또는 특정값 목록으로만 한다. 
	-----> 연산자는 between, in
	-----> 값을 검사하는 것이 목적이다. 		
 * -------------------------
 * 4) 기본키 : primary key (pk) . 기본키 값을 이용하면 테이블의 행을 식별(구별)  
 * 	     where 기본키컬럼=값;	조건실행 했을 때 1개의 행 조회됩니다.
 * 			 자동으로 인덱스(색인)(찾아보기)로 생성이됩니다. 
 * 			 인덱스는 검색(조회) 속도 향상시키는 방법입니다.
 * 	     기본키는 NOT NULL 과 UNIQUE 2가지 제약조건에 해당합니다.
 * 			 기본키는 테이블에서 1개만 가능합니다. 
 * 			 단, 기본키는 구성하는 컬럼은 1개 이상 (2개,3개...) 가능합니다.
 * 
 * 5) 외래키 : 테이블 간의 참조를 위해서 사용합니다.
 * 			 외래키는 테이블 1개에서 여러 개 만들어질 수 있습니다.
