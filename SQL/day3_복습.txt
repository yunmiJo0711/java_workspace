관계형 데이터베이스
- 데이터를 저장해야할 개체와 관계를 구별해야 합니다.
- 예시 : 성적관리 
	ㄴ 개체 : 학생, 교사, 교과목(데이터 주체)
	ㄴ 관계 : 성적 ( 개체 사이에서 발생하는 데이터. 관심주제)

- 개체와 관계 모두 테이블에 데이터를 저장합니다.

- 외래키 
	ㄴ 관계 데이터를 저장하는 테이블에서
                 개체의 데이터를 참조하는 것.
	ㄴ 문제 1 : 도서, 저자 테이블
	            저자_ID 속성은 저자 개체 속합니다.
                         저자_ID 속성은 도서테이블에서 참조합니다.

	ㄴ 문제 2 : 쇼핑몰에서 관심 주제는 구매(판매)
		 개체는 관심 주제 데이터를 만들어 냅니다.
		 개체 : 회원, 상품
	              관계 : 회원이 언제, 몇개,얼마에 어떤 상품 구매?
		          테이블명은 구매(또는 판매)
                           구매 테이블의 외래키 : 회원ID 와 상품ID
